# Internal Working of javac in Java

## Introduction
`javac` is the Java compiler used to convert Java source code (`.java`) into bytecode (`.class`). The generated bytecode is later executed by the JVM (Java Virtual Machine).

Command:

```bash
javac Hello.java
```

---

# Complete Internal Flow of javac

```text
Source Code (.java)
        ↓
Lexical Analysis
        ↓
Syntax Analysis
        ↓
Semantic Analysis
        ↓
Abstract Syntax Tree (AST)
        ↓
Optimization
        ↓
Bytecode Generation
        ↓
.class File
```

---

# Example Java Program

```java
public class Hello {
    public static void main(String[] args) {
        int a = 10;
        int b = 20;

        System.out.println(a + b);
    }
}
```

---

# 1. Lexical Analysis

The compiler reads the source code character by character and converts it into tokens.

## Example

```java
int a = 10;
```

Converted into tokens:

```text
[int] [a] [=] [10] [;]
```

## Types of Tokens

| Token | Type |
|---|---|
| int | Keyword |
| a | Identifier |
| = | Operator |
| 10 | Literal |
| ; | Separator |

---

# 2. Syntax Analysis

The parser checks whether the program follows Java grammar rules.

The compiler creates an internal structure called:

# AST (Abstract Syntax Tree)

## Example

```java
int a = 10 + 20;
```

AST Representation:

```text
      =
     / \
    a   +
       / \
     10  20
```

---

# 3. Semantic Analysis

The compiler checks the meaning of the code.

It verifies:

- Variable declarations
- Method existence
- Datatypes
- Access modifiers
- Inheritance rules

## Example Error

```java
int a = "Hello";
```

Compiler Error:

```text
Type mismatch
```

Because String cannot be stored inside an integer variable.

---

# 4. Symbol Table Creation

The compiler creates internal tables to store information about:

| Item | Information Stored |
|---|---|
| Variables | Name and datatype |
| Methods | Return type and parameters |
| Classes | Fields and methods |

---

# 5. Annotation Processing

If annotations exist in the program, the compiler processes them.

## Example

```java
@Override
```

Frameworks like Spring and Lombok use annotation processing heavily.

---

# 6. Optimization Phase

The compiler improves code efficiency.

## Example

```java
int x = 10 + 20;
```

Internally optimized to:

```java
int x = 30;
```

This optimization is called:

# Constant Folding

---

# 7. Bytecode Generation

The compiler converts the Java code into JVM bytecode.

## Example

```java
System.out.println("Hello");
```

Generated bytecode (simplified):

```text
getstatic
ldc
invokevirtual
return
```

The bytecode is stored inside:

```text
Hello.class
```

---

# Inside the .class File

A `.class` file contains:

| Component | Purpose |
|---|---|
| Magic Number | Identifies Java class |
| Constant Pool | Stores constants |
| Methods | Bytecode instructions |
| Fields | Variables |
| Metadata | Class information |

---

# Magic Number

Every Java class file starts with:

```text
CAFEBABE
```

This tells the JVM that the file is a valid Java class file.

---

# JVM Execution After Compilation

When you run:

```bash
java Hello
```

The JVM performs:

1. Class Loading
2. Bytecode Verification
3. Machine Code Conversion
4. Program Execution
